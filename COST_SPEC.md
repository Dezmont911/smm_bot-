# COST_SPEC — контроль расходов (личный/админ)

Постановка для Claude Code. Цель: кнопка **«💰 Расходы»** в меню — видна ТОЛЬКО владельцу
(`cfg.ADMIN_CHAT_ID`), показывает траты по всем платным сервисам бота. Тестеры её не видят.

Связано с `SAAS_SPEC.md` (таблица `usage`) и пунктом «Учёт расходов» в `BACKLOG.md` (Tier 2) —
делать заодно: один учёт кормит и капы тестеров, и личный дашборд.

---

## 1. Что реально стоит денег

| Сервис | Тип | Как считать |
|---|---|---|
| Anthropic Claude (LLM) | переменный | по ТОКЕНАМ (input+output) × цена модели. Все вызовы: генерация, рефраз, web_search-темы, classify, image-query, дискавери лент |
| fal.ai FLUX (картинки) | переменный | за изображение (~$0.003/шт) × кол-во |
| Webshare прокси | фикс. (подписка) | справочно из `.env` (мес. сумма) |
| VPS | фикс. (мес.) | справочно из `.env` |
| Pexels / Unsplash | бесплатно | показываем кол-во запросов, $0 |
| Telegram Bot/Telethon | бесплатно | — |

Вывод: точный учёт нужен по **Claude (токены)** и **FLUX (штуки)**. Остальное — фикс/инфо.

---

## 2. Учёт (единые точки)

- **Claude:** единственный чокпоинт — `claude_helper.claude_text`. Снимать из ответа
  `response.usage` (`input_tokens`, `output_tokens`) + модель и писать в учёт. Это даёт
  ТОЧНУЮ стоимость, а не оценку по числу вызовов.
  - Добавить в `claude_text(...)` необязательный параметр `purpose` (generation / rephrase /
    topic_search / classify / image_query / discovery) — для разбивки по функциям.
- **FLUX:** `image_generator.generate_image` — инкремент на каждую успешную генерацию.
- (опц.) прокидывать `channel_id`/`owner_id`, чтобы потом видеть расход по каналам/тестерам.

### Таблица
```
usage_costs(
  day        TEXT,         -- 'YYYY-MM-DD' (UTC)
  service    TEXT,         -- 'claude' | 'flux' | 'pexels' | 'unsplash'
  model      TEXT,         -- для claude: имя модели; иначе ''
  purpose    TEXT,         -- generation/rephrase/topic_search/... ('' если не задан)
  calls      INTEGER DEFAULT 0,
  in_tokens  INTEGER DEFAULT 0,
  out_tokens INTEGER DEFAULT 0,
  PRIMARY KEY (day, service, model, purpose)
)
```
($ не храним — считаем на лету из токенов × прайс, чтобы при смене цен пересчитывалось.)
Миграция — идемпотентный CREATE в `database.py` + `deploy_fixes`. Старые данные не нужны.

### Прайс (в `.env`/`config.py`, редактируемо)
```
PRICE_CLAUDE_IN_PER_MTOK   = 0.80   # $ за 1M input-токенов (под текущую модель Haiku)
PRICE_CLAUDE_OUT_PER_MTOK  = 4.00   # $ за 1M output-токенов
PRICE_FLUX_PER_IMAGE       = 0.003
FIXED_COST_PROXY_MONTH     = ...    # справочно
FIXED_COST_VPS_MONTH       = ...    # справочно
```
(числа — плейсхолдеры; поставить реальные из тарифов. Все $ в дашборде помечать как «оценка».)

## 3. Кнопка «💰 Расходы» (меню, только админ)

- В главном меню (или в админ-разделе) пункт виден, если `user_id == ADMIN_CHAT_ID`.
- Экран показывает за **Сегодня / 7 дней / 30 дней** (переключатели):
  - **Claude:** вызовы, input/output токены, **оценка $** (по моделям).
  - **FLUX:** штук, **оценка $**.
  - **Сток (Pexels/Unsplash):** запросов, $0.
  - **ИТОГО переменных $** за период.
  - Блок «Фиксированные (справочно)»: прокси/VPS из `.env`.
- (опц.) разбивка по `purpose` (на что уходит LLM: генерация vs рефраз vs поиск vs дискавери)
  и топ-каналов по расходу — полезно для оптимизации.

## 4. Края / заметки

- Числа $ — ОЦЕНКА (прайс из конфига может отставать от реального биллинга). Подписать.
- Если `response.usage` недоступен (ретрай/ошибка) — считать вызов, токены 0 (не падать).
- Учёт не должен ронять генерацию: запись в `usage_costs` обернуть в try/except.
- Это личный инструмент владельца; для тестеров — НЕ показывать (тот же гейт, что и /admin).

- [ ] `claude_text` пишет токены+модель(+purpose) в `usage_costs` на каждый вызов.
- [ ] FLUX-генерация инкрементит `usage_costs(service='flux')`.
- [ ] Кнопка «💰 Расходы» видна только `ADMIN_CHAT_ID`, тестерам — нет.
- [ ] Показывает Claude (токены+$), FLUX (шт+$), сток ($0), ИТОГО за день/7/30 + фикс. справочно.
- [ ] Сбой учёта не влияет на генерацию/постинг (try/except).
