# 📘 Инструкция: ИИ Контент-Завод для 45 Telegram каналов

> **Статус:** Внутренний техдок для проектирования и сборки  
> **Цель:** Заменить людей-контентщиков и постеров на автоматизированный pipeline  
> **Ключевая боль:** Человеческий фактор → пропущенные посты → рекламный пост не перекрыт → деньги потеряны

---

## 🔑 Главная логика системы

```
Яндекс РСЯ выпустил рекламный пост
         ↓
Бот-мониторинг засёк событие
         ↓
Берёт готовый пост из буфера
         ↓
Публикует через 5–15 минут (имитация человека)
         ↓
Буфер пополняется автоматически
```

**Золотое правило:** Буфер никогда не должен опустеть. Минимум — 8 постов на канал (запас на 2 дня).

---

## 🏗️ Архитектура: 5 слоёв

```
[1. База знаний каналов]
         ↓
[2. Генератор контента (LLM)]
         ↓
[3. Буфер / Очередь постов]
         ↓
[4. Мониторинг рекламных постов]
         ↓
[5. Постер-бот → Telegram]
         ↑
[6. Дашборд + Алёрты]
```

---

## Слой 1 — База знаний каналов

**Что это:** JSON/БД карточка для каждого из 45 каналов.  
**Зачем:** Это системный промпт для генерации — без него контент будет безликим.

### Структура карточки канала

```json
{
  "channel_id": "@mychannel",
  "name": "Финансы для людей",
  "topic": "личные финансы, инвестиции, сбережения",
  "audience": "25–45 лет, офисные работники, хотят разобраться в деньгах",
  "tone": "дружелюбный эксперт, без снобизма",
  "post_length": "100–200 слов",
  "use_emoji": true,
  "forbidden_topics": ["крипта", "MLM", "займы МФО"],
  "post_formats": ["совет дня", "факт", "вопрос аудитории", "мини-разбор", "статистика"],
  "example_posts": [
    "Знаете, почему 80% людей не копят? ...",
    "Сколько реально нужно откладывать? ..."
  ],
  "rss_sources": [
    "https://rbc.ru/finances/rss",
    "https://vc.ru/finance/rss"
  ]
}
```

**Действие:** Создать карточки для всех 45 каналов. Это делается один раз вручную, потом система живёт сама.

## Слой 2 — Генератор контента

**Что делает:** Каждое утро в 6:00 генерирует 10–12 постов для каждого канала.

### Алгоритм генерации

```
1. Взять карточку канала
2. Спарсить RSS-ленты по теме → получить 5–10 свежих инфоповодов
3. Выбрать 4 инфоповода (по score: новизна + релевантность аудитории)
4. Для каждого: сгенерировать 2 варианта поста
5. Авто-выбор лучшего варианта (длина, наличие хука, CTA или вопроса)
6. Проверка на анти-повтор (косинусное сходство >80% → регенерация)
7. Положить в буфер
```

### Источники тем (приоритет)

| Приоритет | Источник | Примечание |
|---|---|---|
| 1 | RSS по теме канала | Свежие инфоповоды |
| 2 | Банк "вечнозелёных" тем | Всегда актуально |
| 3 | Сезонный календарь | Праздники, события |
| 4 | Переработка старых постов | Посты 6+ месяцев назад |

### Форматы постов (ротация)

Система чередует, чтобы контент не был однообразным:
- ✍️ **Совет/инструкция** — как сделать X
- 📊 **Факт/цифра** — удивительная статистика
- ❓ **Вопрос аудитории** — вовлечение
- 🔍 **Мини-разбор** — объяснение темы
- 📅 **Инфоповод** — новость + наш взгляд

**Правило:** Если 2 поста подряд одного формата → следующий обязательно другого.

### Промпт-структура (системный промпт для LLM)

```
Ты — редактор Telegram-канала "{name}".
Тема: {topic}
Аудитория: {audience}
Тон: {tone}
Длина поста: {post_length}
Формат: {выбранный формат}
Запрещено упоминать: {forbidden_topics}

Инфоповод: {новость из RSS}

Напиши пост. Только текст поста, без комментариев.
```

## Слой 3 — Буфер и очередь

**Что это:** Готовые посты, ожидающие публикации.

### Структура записи в буфере

```json
{
  "post_id": "uuid",
  "channel_id": "@mychannel",
  "content": "текст поста",
  "format": "совет",
  "generated_at": "2025-01-15T06:12:00Z",
  "status": "ready",
  "similarity_score": 0.12
}
```

### Правила буфера

- **Минимум:** 8 постов на канал (на 2 дня вперёд)
- **Комфорт:** 12–16 постов (на 3–4 дня)
- **Экстренная генерация:** если < 4 постов → запустить генерацию немедленно + алёрт
- **Критический уровень:** если < 2 постов → алёрт администратору + генерация в режиме максимального приоритета

### Когда пополняется

- ⏰ Плановая генерация: каждое утро в 06:00
- 🚨 Экстренная: при падении буфера ниже порога
- ✋ Ручная: через дашборд (если нужно добавить особый пост)

---

## Слой 4 — Мониторинг рекламных постов Яндекса

**Это самый критичный компонент.** Именно он запускает публикацию.

### Логика работы

```python
# Псевдокод мониторинга
every 3 minutes:
    для каждого канала:
        получить последние 5 постов через Telegram API
        если найден пост от Яндекс РСЯ:
            если он ещё не обработан (нет в таблице processed_ads):
                записать в processed_ads
                поставить задачу: опубликовать следующий пост через random(5, 15) минут
```

### Как определить пост Яндекса

Рекламные посты Яндекс РСЯ имеют характерные признаки:
- Sender ID / Bot ID рекламной сети
- Метка `#реклама` или `Реклама` в тексте
- Forwarded from рекламного аккаунта

**Действие:** Нужно вручную зафиксировать паттерн для ваших конкретных каналов и прописать в фильтре.

### Таблица отслеживания

```sql
CREATE TABLE processed_ads (
  id UUID PRIMARY KEY,
  channel_id TEXT,
  ad_message_id BIGINT,
  detected_at TIMESTAMP,
  response_post_id UUID,        -- какой пост был опубликован в ответ
  published_at TIMESTAMP,
  status TEXT                   -- detected / published / failed
);
```

### Защита от сбоев

| Ситуация | Действие |
|---|---|
| Рекламный пост висит >2 часов без ответа | Алёрт + экстренная публикация |
| Буфер пуст в момент триггера | Экстренная генерация → публикация |
| Telegram API недоступен | Retry каждые 2 минуты, алёрт после 5 попыток |
| Генерация LLM упала | Взять пост из "вечнозелёного" банка |

---

## Слой 5 — Постер-бот

**Что делает:** Публикует посты в Telegram по триггеру из Слоя 4.

### Логика публикации

```
Получен сигнал: опубликовать пост для @channel
    ↓
Взять верхний пост из буфера (статус: ready)
    ↓
Подождать random(5, 15) минут  ← имитация человека
    ↓
Опубликовать через Telegram Bot API
    ↓
Обновить статус поста → published
    ↓
Обновить processed_ads → published
    ↓
Проверить уровень буфера → если < 4, запустить генерацию
```

### Важные настройки постера

- **Задержка публикации:** 5–15 минут после рекламного поста (случайная, не фиксированная)
- **Формат публикации:** Markdown (если есть форматирование в посте)
- **Ошибки:** 3 ретрая с экспоненциальной задержкой, затем алёрт

## Слой 6 — Дашборд и алёрты

### Дашборд: что видно по каждому из 45 каналов

| Канал | Буфер | Рекл. сегодня | Перекрыто | Статус |
|---|---|---|---|---|
| Финансы | 14 постов | 4 | 4/4 | ✅ |
| Авто | 3 поста | 4 | 3/4 | ⚠️ |
| Кулинария | 1 пост | 2 | 2/2 | 🔴 буфер низкий |

**Цветовые статусы:**
- ✅ Зелёный: буфер > 8, все рекламные посты перекрыты
- ⚠️ Жёлтый: буфер 4–8, или есть неперекрытый пост < 1 часа
- 🔴 Красный: буфер < 4, или рекламный пост не перекрыт > 2 часов

### Система алёртов (в Telegram-бот администратора)

```
🔴 КРИТИЧНО: @channel_name — буфер пуст! Генерация запущена.

⚠️ ВНИМАНИЕ: @channel_name — рекламный пост от 14:32 
не перекрыт уже 90 минут. Публикуем сейчас.

✅ ИНФО: Утренняя генерация завершена. 
Создано 480 постов для 45 каналов. Буфер в норме.
```

### Ручные действия через дашборд

- Добавить пост в очередь вручную
- Пропустить / удалить пост из буфера
- Запустить внеплановую генерацию для канала
- Изменить карточку канала
- Посмотреть историю публикаций

## 🛠️ Технический стек

| Компонент | Выбор | Обоснование |
|---|---|---|
| **Язык** | Python 3.11+ | Telethon, богатая экосистема |
| **Генерация контента** | Claude API (claude-sonnet) | Лучшее качество текста на русском |
| **Мониторинг Telegram** | Telethon | Чтение каналов как пользователь |
| **Постинг** | Telegram Bot API | Стандартный способ публикации |
| **Очередь задач** | Redis + Celery | Надёжно, просто, работает |
| **База данных** | PostgreSQL | Основная БД: посты, каналы, статусы |
| **Планировщик** | Celery Beat | Утренняя генерация в 06:00 |
| **RSS парсер** | feedparser | Простой и надёжный |
| **Алёрты** | Telegram Bot API | Пуш в чат администраторов |
| **Хостинг** | VPS (4 CPU / 8GB RAM) | Достаточно для 45 каналов |
| **Дашборд** | n8n UI или FastAPI + простой фронт | Быстрый старт |

### Схема базы данных (основные таблицы)

```sql
-- Карточки каналов
channels (id, tg_handle, name, topic, tone, config_json, active)

-- Буфер постов
posts (id, channel_id, content, format, status, generated_at, published_at)

-- Рекламные события
processed_ads (id, channel_id, ad_message_id, detected_at, response_post_id, status)

-- Банк вечнозелёных тем
evergreen_topics (id, channel_id, topic, last_used_at, use_count)

-- Лог ошибок
error_log (id, channel_id, error_type, message, occurred_at, resolved)
```

## ⚠️ Точки отказа и как их закрыть

| Риск | Вероятность | Решение |
|---|---|---|
| Claude API упал | Низкая | Fallback на OpenAI GPT-4o |
| Буфер опустел | Средняя | Экстренная генерация + алёрт |
| Telegram API rate limit | Средняя | Очередь с задержками между запросами |
| Паттерн рекламы Яндекса изменился | Средняя | Гибкий фильтр + алёрт на нераспознанные посты |
| VPS упал | Низкая | Docker restart + мониторинг аптайма |
| Сгенерированный пост плохого качества | Средняя | Few-shot примеры + рейтинг постов |

## 💡 Фичи второй очереди (после MVP)

1. **A/B тест форматов** — система считает охваты и выбирает лучший формат для каждого канала
2. **Авто-обновление банка тем** — раз в неделю парсит топ-посты в нише
3. **Сезонный контент** — праздники и события подтягиваются автоматически
4. **Мультиканальность** — один пост адаптируется под несколько каналов одной тематики
5. **Аналитика** — подключить TGStat или Telemetr API для отслеживания реакций
6. **Картинки к постам** — генерация обложек через FLUX/DALL-E для постов с изображением
7. **Контент-план** — возможность задать тему недели для канала вручную

---

## 📌 Ключевые принципы проекта

> **1. Буфер — это жизнь.** Система должна сделать всё возможное, чтобы буфер не опустел.

> **2. Лучше опубликовать средний пост, чем не опубликовать ничего.** У нас всегда есть вечнозелёный резерв.

> **3. Каждое действие логируется.** Любой сбой должен быть воспроизводим.

> **4. Система сама чинит себя.** Алёрт — это исключение, не норма.
