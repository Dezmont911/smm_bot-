## Tier 1 — качество контента (главная боль из логов + критика V2)

- [ ] **Эмбеддинг-гейт релевантности темы** (он же «Topic Rating»): косинус кандидат-темы
      к профилю канала, отсев off-topic ДО генерации. Дёшево (reuse `dedup`), бьёт по
      дрейфу. Обязательно с in-scope fallback (не словить пустой буфер).
- [ ] **Источник под тип канала**: релиз → референсы; новости → search; вечнозелёное →
      evergreen/RSS. Сделать осознанным выбором/подсказкой при `/add`, а не ручной правкой.
- [ ] **Quality Check как явный слой**: длина + запретные слова + дубль + отказ
      (детерминированно) + максимум **1** регенерация при провале. Куски есть — собрать.

## Tier 2 — масштаб и бюджет (40–100 каналов)

- [ ] **Пуллинг тем по нишам**: один web-search на нишу → раздать N каналам. Главный
      выигрыш по LLM-расходам.
- [ ] **Учёт расходов**: лог токенов/вызовов/FLUX в день по каналам + дневной порог + алерт.
- [ ] **Лимитер LLM/Telegram** (семафор) + джиттер публикаций внутри часа.
- [ ] **`/bulk_add`** — массовое добавление списком (онбординг десятков каналов).
- [ ] **Папки/группы каналов** + групповые действия.

## Tier 3 — управление и надёжность

- [ ] Режим модерации на канал (авто-публикация vs ревью).

- [ ] Идемпотентность публикации (анти-двойной-постинг при падении между send и mark_published).
